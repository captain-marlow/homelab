The local tier is better aimed at cheap, narrow work (heartbeat, classification, routing hints), not the planner. Revisit only with a GPU + a much larger open model.

- **Hermes ACP in VS Code** — wire Hermes into VS Code via its ACP integration, so the Mac-side executor is reachable from the editor. Pairs with the Hermes track (P006). <https://hermes-agent.ai/integrations/vscode>
- **Pin the Ollama LXC to performance (P) cores** — pve01's CPU is a hybrid P-core/E-core design, so a raw vCPU/thread count is a poor proxy for speed: llama.cpp inference threads that land on slower E-cores drag latency and make it inconsistent. Pin CT172 to P-core host CPUs via `lxc.cgroup2.cpuset.cpus` in `/etc/pve/lxc/172.conf` (identify which host CPU IDs are P-cores first, e.g. `lscpu -e`). Tune alongside the Ollama-tier core-count work; applies to any latency-sensitive local inference.

- **OpenClaw "prose"** — explore further: <https://docs.openclaw.ai/prose>
- **Ansible auto-registers new LXCs in Mac `~/.ssh/config`** — every LXC created via the Mac Ansible flow should append a `Host` block (alias / `HostName` / `User` / `IdentitiesOnly yes` / `IdentityFile`) matching the existing uniform pattern, so new containers are reachable by name immediately. A `blockinfile` task keyed on the alias. Belongs with the Ansible/Hermes repo-integration work (P006).
- **Ansible LXC baseline-hardening role** — every new LXC should come up hardened by default: push the Mac's public key into the container (`authorized_keys` for the admin user) for immediate key-based access; **disable root SSH login** (`PermitRootLogin no`) and password auth (key-only); create a non-root sudo user; plus a general pass (firewall, unattended security upgrades). A reusable `baseline`/`hardening` role applied to every container. Pairs with the SSH-config auto-add above; same Ansible/Hermes track (P006).
- **Read-only web ingest for the `architect` agent** — give the planner a web-fetch HTTP GET and/or web-search tool. A planner that cannot research while planning is half-blind; “read-only” should mean “can ingest, cannot mutate,” not “cannot learn.” A GET-only fetch is a read, not a system change, so it preserves the planner → executor → human-gate separation. Constraints: GET-only; no POST, write, or auth-bearing methods; no exec; treat all fetched content as untrusted prompt-injection surface; scope the capability exclusively to the architect tool allowlist. Source: arose while researching agentskills.io, which architect could not fetch. **Now also the home for restoring web search generally:** OpenClaw's built-in web search was Gemini-backed (Generative Language API) and is **disabled as of 2026-06-25** (Gemini removed — 0-token key; the `google-gemini-cli` OAuth path was declined for Google account-suspension risk). So a **non-Gemini search provider (Tavily / Brave / Exa — real free tiers)** would serve *both* the architect's read-only ingest and general agent web search in one move. Pick a provider with a clean API key (no OAuth-impersonation risk). **→ The architect read-only-browse half is now promoted to P009; the SearXNG bullet below is the self-hosted keyword-search layer for later.**
- **OpenAI Daybreak — securing the world** — explore further: <https://openai.com/index/daybreak-securing-the-world/>
- **Documentation MCP for agents (e.g. Context7)** — give the homelab agents (Hermes/OpenClaw) an MCP server that serves up-to-date, version-specific library/framework docs on demand, so code and configs are written against current APIs instead of stale training data. Context7 (Upstash) is one candidate: <https://github.com/upstash/context7>. Pairs with Hermes's MCP support (`hermes mcp add`) and the docs track (P007/P008).
- **SearXNG (self-hosted metasearch) for the homelab** — stand up a SearXNG instance (small LXC/docker via the `proxmox_lxc_docker_host` role, same pattern as Ollama/Synapse) as a **keyless, self-owned keyword-search backend** for the agents. OpenClaw supports `searxng` as a `web.search` provider (verified in the dist alongside tavily/exa/duckduckgo; *not* brave). No external account, no API key, **zero account-suspension risk** — the clean answer after declining the Gemini OAuth path (0-token key + Google ban risk). Covers "find unknown sources" once browse/scrape (P009) isn't enough. Ollama-tier-sized. Quick alternative if impatient: a Tavily free key (1000/mo).

Only worth opening if autonomous continuous planner↔executor loops (no human relay) become a concrete goal.
